# ASCII Terraria 开发路线图

## 设计原则
- 俯视图 roguelike，x/y 双向无限世界
- 每个新系统必须和至少一个现有系统产生非预期的交互（涌现）
- 数据字段必须在加入时就明确消费方，不留死数据
- 先验证机制对不对，再堆数量

## 参考神作的核心借鉴
| 游戏 | 借鉴什么 | 不做什么 |
|------|---------|---------|
| CDDA | 生存资源管理、数据驱动 | 载具系统（暂不做） |
| 矮人要塞 | 系统涌现、规则碰撞 | 间接操控（不做） |
| Elona | 技能靠使用升级、数值成长曲线 | 完全随机事件（暂不做） |
| NetHack | 回合制、确定性生成 | 物品鉴定/诅咒（不做） |
| Terraria | 方块破坏与建造、装备槽位 | 侧视图/重力（不做） |

## 明确不做
- 物品实例化（独立ID + 鉴定/诅咒状态）——保持堆叠模型
- 矮人要塞式间接管理——保持 @ 直接操控
- NetHack 式信息不对称（未鉴定物品博弈）
- 侧视图/重力——正式定位为俯视图
- 饥饿系统——已移除，简化生存体验
- 逐格光照计算——已移除，保留全局昼夜色调
- 装备耐久度——已移除，减少负反馈

## 暂缓/不做
| 项目 | 原因 |
|------|------|
| 物品实例化（UUID/鉴定/诅咒） | 明确不做，保持堆叠模型 |
| 载具系统 | 暂不做 |
| 矮人式间接管理 | 不做 |
| 流体物理（水/岩浆/沙子） | 暂缓，性能开销大 |
| 行动力系统（AP/Energy） | 暂缓，当前无攻速差异化需求 |
| FoV 阴影投射光照 | 不做，保留全局昼夜色调 |
| EventBus 消息总线 | 暂缓，单线程回合制不需要异步解耦 |
| Numpy 加速地形生成 | 暂缓，移动端纯 Python 足够流畅 |

## 操作说明
| 按键 | 功能 |
|------|------|
| hjkl / 方向键 | 移动/攻击/挖掘 |
| d + 方向键 | 定向挖掘（拆除建筑/尸体） |
| c | 合成菜单 |
| e | 装备菜单 |
| . | 重复上次建造 |
| x | 查看模式（移动光标查看远处） |
| 回车 | 建造模式下放置 |
| S | 存档 |
| L | 读档 |
| r | 重载JSON数据 |
| q | 退出 |
